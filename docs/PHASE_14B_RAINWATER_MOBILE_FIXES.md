# Phase 14B – Regenwasser Mobile/Event Stabilisierung

- Select-Felder im Regenwasser-Modul rendern verzögert, damit native Mobile-Selects nicht durch einen Full-Render erneut geöffnet werden.
- Speichern/Aktualisieren committen vor der Aktion alle aktuellen Feldwerte in den Store.
- Die Speicher-Card ist wieder Teil des Regenwasser-Layouts.
- Mobile Modulnavigation wird bei aktiver Tastatur ausgeblendet, damit sie nicht über die Tastatur geschoben wird.
- Ziel: Regenwasser folgt weiter dem Store-first/Event-Pipeline-Muster des Referenzmoduls Heizung/Kälte.

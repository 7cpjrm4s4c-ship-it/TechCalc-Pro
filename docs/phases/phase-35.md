### Phase  35F SAVE DIALOG MODULE CLEANUP

Druckhaltung, Schmutzwasser und Regenwasser wurden auf das Single-Controller-Pattern der funktionierenden Referenzmodule ausgerichtet. Die Speicherlogik darf nicht mehr parallel in Modul-Controller-Objekten und in `createLineSectionController()` existieren. Pro Modul ist ein zentraler Save-Controller zulässig.

Key sections: Ziel; Entscheidung; Änderungen; Verifikation; Hinweis.

### Phase  35D SAVE DIALOG PARITY

Die Speicherdialoge von Druckhaltung, Schmutzwasser und Regenwasser werden nicht mehr als Sonderfälle behandelt. Sie verwenden denselben Line-Section/Saved-Record-Workflow wie die stabilen Referenzmodule Rohr und Pufferspeicher. Manuell prüfen: 1. Druckhaltung: Anlage speichern, auswählen, aktualisieren, löschen.

Key sections: Ziel; Änderungen; Verifikation; Retest-Priorität.
